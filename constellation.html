Constellation

The organs that make up the custody stack, the planes they belong to, and compositions that illustrate how they can work together. Every organ is independently usable. None requires another.

NQ does not require agent_governor. Agent_governor does not require NQ. The monitoring lane that wires them — NQ witnesses a disk state, Nightshift defers and reconciles the governed work, agent_governor refuses the action if the evidence is stale — is a natural composition and one illustration. It is not the product. Use the organ you need.

Honesty markers: wired = implemented and tested; specified = ratified contract, implementation pending. Resolved ≠ supported. Maturity grades are per-organ; no grade inflation. Nothing on this page is armed for live cage execution — that refusal is by construction. See limits.

Organs grouped by what kind of work they do. A plane is analytical — it says what role the organ plays, not what it depends on.

Illustrated composition (not a prerequisite stack): nq (witness) → nightshift (defer + reconcile) → agent_governor (govern). Each organ in that lane works standalone — absent any neighbor, the seam yields honest absence or cannot-testify, never silent failure.

Witness / testimony plane

Observe and report. These organs produce evidence; they do not gate authority. Cannot-testify is a first-class outcome, not an error.

nq flagship
handles
Observation-to-claim preflight for operational systems. Witnesses disk health, service state, host metrics, and similar substrate facts; packages each finding with its evidence so the claim is traceable to an observation. When evidence is unavailable, emits cannot-testify rather than a smooth nothing — the refusal is the product surface, not an error to handle.
refuses
Producing a finding it cannot trace to an actual observation; smoothing over evidence loss.
grade
Operational — public GitHub, Apache-2.0, 1203 tests, 15+ operator docs. Operator calibration 2026-07-05: “usable today by a normal SRE.” (Campaign STATUS.md §Sprint 3, packet 9b.)
porter v0.1
handles
Substrate courier. Executes work via ssh, serial, or recipe substrates and packages the outcome as a record.v0 with exit-code custody. The record carries whether the exit code was actually observed; a record with exit_code_observed: false is refused, not promoted.
refuses
Marking work complete when exit_code_observed is false; emitting outcome vocabulary that belongs to the caller’s domain (porter’s outcomes name transport facts, not admissibility verdicts).
grade
v0.1 — three substrates (ssh, serial, recipe), 19 tests, charter commitments closed. demo/refused-exit.sh produces a live refusal specimen. Serial substrate may slip to v0.2. (STATUS.md §Lane P.)
verifier operational
handles
Formal constraint checking via a Z3 sidecar. Accepts a declared invariant set and a proposal; refuses proposals that violate the constraints and emits a verdict receipt. The specimen at the front of the README is the stale-standing denial.
refuses
Proposals that fail the declared invariants; constraint-checking without a named clock basis on temporal predicates.
grade
Operational — lib, CLI, and MCP server; specimen pinned to README. (Campaign §3 table: READY.)

Authority plane

Gate work. These organs decide whether a claim, intent, or permission may become a premise for action, and emit receipts for the decisions they make.

agent_governor alpha
handles
Admissibility kernel for AI coding agents. Every proposed file write and command requires a typed claim with pointers; the governor verifies and produces receipts. Language is a proposal, not an authority — an agent saying “done” is not the operator approving it. Supervised runs, refusal gallery, governed workflow, and specimen corpus are runnable from a fresh clone.
refuses
Prose as authority; tests-passing treated as code-ready-to-land; permissions exercised after their observation horizon; a rehearsal that confers operational effect.
grade
2.8.1-alpha — ~14.6k tests, demo path fresh-clone verified. Bootstrap limits: in-process custody forgeability, operator-fiat standing, live cage unarmed by construction. (STATUS.md §Sprint 2, exit ticket.)
wicket operational
handles
Single-call admissibility preflight for intents. Decides whether an actor’s proposed action may become a premise before any effect runs; refuses intents from revoked or out-of-scope actors. One call in, one typed verdict out.
refuses
Intents where the actor cannot demonstrate standing or scope; revoked actors.
grade
Operational — SPEC v0.3, 21 fixtures, wicket-guard absorbed and tombstoned. (Campaign §3 table: READY.)
standing operational
handles
Permission observation with observation-age receipts. Produces standing receipts that carry when the permission was seen and when it expires; the gap between check and action is measured on a named clock basis, not wall time. A permission valid at check is not automatically usable at spend time.
refuses
Attesting a permission it cannot verify; a spend whose standing observation lapsed past its horizon by exercise time (standing_before_spendability_not_bounded).
grade
Operational — specimen flow runs; HMAC-only identity substrate (named limit; mTLS/OIDC deferred). (Campaign §3 table: READY.)
linear_accountant reference
handles
Conserved capacity and spend — the reference boundary that ensures no organ mints authority from an ALLOW verdict. An authorized action still refuses without capacity usable at action time. Frozen as a reference seam until a consumer trigger; the transport layer is deliberately deferred.
refuses
Minting capacity from an ALLOW verdict (the organ named consumer, not minter). The no-mint invariant is the load-bearing property; no transport is built until it has a consumer.
grade
Reference — pushed, frozen by design. “Respect the freeze” is the standing operator ruling. (Campaign §3 table: READY as reference seam, not product.)

Operator plane

Operator-facing surfaces over the same daemon. Maude is a TUI; gov-webui is a browser path. Both consume the governed shell contract; neither is a hard requirement for the organs they expose.

maude operational
handles
Operator shell for supervised agent sessions. Runs an AI coding agent under agent_governor; every tool call appears in the decision queue before it runs. When the session ends, the operator reviews the diff and either keeps or discards the changes. Governed-plan runs ingest a plan.md, verify citations, and report as a ReviewPacket.
refuses
Silently promoting session output — promotion is an explicit operator act; the report surface says “not recorded” rather than inferring; used > granted renders OVERRUN, never “authorized.”
grade
v2.4.0 operational — 301 tests, supervised loop, governed-plan path, M-4 run report, live-daemon smoke verified. (STATUS.md §Sprint 4, packet 15.)
gov-webui alpha
handles
Browser-based operator mode over the governor daemon. Legacy chat modes plus desk mode v0 in progress — decision queue, sessions board, and promotion keep/discard, consuming the same shell contract as maude so the two surfaces cannot drift on the wire protocol. Zero-install path for visitors who won’t run a terminal app.
refuses
N/A (operator UI surface; the one-mutation-door rule is enforced in the daemon, not the UI).
grade
v0.5.0 alpha — currency repairs done (U2), DaemonShellClient landed (U3-A, 513 tests); desk routes (U3-B..E) in progress. (STATUS.md §Lanes after S4.)
vscode-governor alpha
handles
Editor surface — wraps the governor check CLI for VS Code. Surfaces security and continuity findings inline. CLI-wrapper architecture, not daemon-dependent.
refuses
N/A (editor integration; verdicts come from the CLI).
grade
Alpha — verified against agent_governor 2.8.1 (2026-07-05, CLI contract; 176 extension tests green); structural extension work is post-MVP. (Campaign §3 table: SMALL compat+smoke packet.)

Memory + read plane

Persist state with receipts; index the corpus for navigation. Neither organ is authority-bearing; both hold things other organs need to find or resume from.

continuity operational
handles
Governed cross-session memory via MCP. Observe → commit → rely with hash-chained receipts; a reliance cannot happen without a prior commit. State mutation without a receipt is refused. Used as the persistent memory substrate for agent_governor sessions.
refuses
State mutation without a receipt; relying on state that was never committed.
grade
Operational — 32 test files, MCP server live. Named limit: reliance-policy semantics are currently permissive (hardening deferred). (Campaign §3 table: READY.)
spine building
handles
Read plane for the constellation — a navigable index over the declared document corpus. Findability is not legitimacy: spine renders an index artifact, never an admissibility verdict. The charter is fixed; a map page may cite spine’s index as “how to browse the corpus” but does not depend on it.
refuses
N/A (observer only; spine holds no authority seam). Output fields must not imply endorsement or status — the non-authority pin test is the stop condition.
grade
Building — S-A documentation de-staled and pushed; core index (S-B) gated on operator decisions OQ-1/OQ-2. (STATUS.md §Lane S after S4.)

Formal plane

Proof artifacts. Not runtime. The theorems prove class boundaries of refusals, not deployment safety.

lean v7.0.0
handles
Formal audit of refusal-class boundaries. Machine-checks that a given refusal kind is required by the custody discipline’s own definitions. Read-only audit surface; not runtime. Citation tiers: [1.0] stable / [annex] compiled / [scratch] reconnaissance. Annex cannot ratify.
refuses
N/A (proof artifact). The proofs refuse to compile if a stated boundary is falsified; they prove the class, not any deployed instance.
grade
v7.0.0 — sorry-free; Zenodo DOI. See limits for what the theorems do and don’t prove. (Campaign §3 table: READY.)

Nightshift

Deferred governed work. Earlier-stage than nq; presented here as a specimen, not a maturity peer.

nightshift specimen
handles
Deferred governed work with receipts and reconciliation. Holds work that arrives outside the authorized window, reconciles it when the window opens, and refuses completion claims without a reconciliation receipt. The stale-witness refusal specimen runs from the README with no live infrastructure.
refuses
Completion claims without reconciliation; work completed with a stale witness.
grade
Specimen — runnable as a demonstration of the stale-witness refusal; earlier-stage than nq as a product. The monitoring lane (nq → nightshift → agent_governor) presents nightshift as the deferred-work layer, not as a maturity peer of nq. (Campaign §3 table: READY as specimen, not as product claim.)

Runnable material

What a stranger can run or inspect today, from a fresh clone of agent_governor.

  • TOUR.md Three-act walkthrough of the demo path: refused spend, receipt interrogation, OPA contrast. Narrates what each receipt field means.
  • GOVERNED_WORKFLOW.md Smallest end-to-end governed workflow: propose → verify → apply → receipts, with zero external agent required. Every command executed; two refusal modes included.
  • REFUSAL_GALLERY.md Eight organs, each refusing in 30 seconds with verbatim output and the receipt field that carries the typed reason. No network, LLM, or live infrastructure required.
  • NON_GRANTS.md What agents are deliberately not granted: no self-approval, no authority from prose, no minting from ALLOW, no agent-to-agent coordination, no unwitnessed clock math. Each entry carries a code pointer and pinning test.
  • specimens/ Self-verifiable specimen corpus: queue latch, approval witness, ReviewPacket, verify-run receipts. README walks digest re-verification by hand.

Quick start: git clone https://github.com/unpingable/agent_governor && cd agent_governor && pip install -e . then ./demo/refused-spend.sh. The demo fails loudly if it passes for the wrong reason.
