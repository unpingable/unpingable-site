James Beck

Modern systems often decide faster than their evidence stays valid.

A permission valid when it was checked gets exercised after it lapsed. A retracted grant lands on a downstream reader that never got the retraction. A verdict logged as “approved” but never sealed.

Custody is the layer that catches those gaps — distinct from authorization, from a run that finished, from a proposal that passed. Small self-hosted tools; proof artifacts on the side.

The custody stack

The custody discipline in parts. The edges are the product; the organs implement them.

Not a policy engine. Not a hosted platform. Not an AI-safety oracle.

Hover an edge for the claim it enforces. Solid shipped; dashed mapped, not built. Drill runs demonstrate structure — they cannot confer operational effect.

Organs — what each handles, and what it refuses

Python libraries and CLIs you self-host. nq is one binary; verifier ships a Z3 sidecar; Lean is for proof artifacts (read-only audit, not runtime). No SaaS, no hosted control plane. Badges: operational in daily use · research exploratory.

Projects

Outside the core stack: supervised runs, field labs, and one-off experiments. Core organs are listed above.

Supervised runs

Where you supervise an AI coding agent through agent_governor.

maude Runs an AI coding agent under agent_governor. You see every action before it happens; approve, deny, or let read-only stuff through. When the run ends, review the diff and either accept the changes or throw them out.
